# 开写/修改任一章节前的必读清单

> 硬性规定：在动笔写新章节或修改已有章节**之前**，必须先完成本清单。未完成不得开始正文写作。

---

## 第一步：读核心规则（5分钟）

必须重新打开并通读一次：

- [ ] `docs/笔法核心规则.md`

重点重温：
1. 一段不超过三句话
2. 不写心理描写，写动作
3. 禁止"不是X，是Y"纠正句式（每章≤3处）
4. 禁用过渡词（然后/一会儿/忽然/接着/之后，每章≤2处）
5. 章末必须有钩子
6. 主角信息边界——他不知道的就是不知道
7. 信息挤牙膏式释放
8. 每章至少三处能力/法宝/修为实战激活
9. 场景之间不写过渡
10. 第一句必须入戏

---

## 第二步：读设定文档（按优先级）

### 写新章节前必读

- [ ] `docs/重构设定_岁蚀全本总纲.md` — 世界观、三条主线、七卷结构
- [ ] `docs/人物档案_重构.md` — 角色设定、阵营、关系
- [ ] `docs/势力划分与部门_重构.md` — 七大势力、部门、出场预设
- [ ] `docs/域外设定与过渡设计.md` — 域外规则、过渡层级
- [ ] `docs/功法与战斗体系.md` — 境界、招式、法宝、阵法
- [ ] `docs/伏笔与线索追踪.md` — 已埋伏笔状态、本章该回收的条目

### 修改已有章节前额外必读

- [ ] 本章原文
- [ ] 前一章原文
- [ ] 后一章原文

---

## 第三步：明确本章任务

用三句话写在本章开头的注释里（写完后删除）：

```
<!--
本章推动：__________
本章改变：__________
本章埋下：__________
章末钩子：__________
本章回收伏笔（来自第X章）：__________
-->
```

必须能回答：
- [ ] 本章推动了什么？
- [ ] 本章改变了什么？
- [ ] 本章为后面埋了什么？
- [ ] 章末钩子是什么？
- [ ] 本章有没有必须现在解决的冲突？

如果答不上来，先回去做大纲，不要动笔。

---

## 第四步：确认主角信息边界

- [ ] 本章出现的新势力/新人物/新术语，主角是如何知道的？（亲眼见/亲耳听/已有知识合理推断/猜错）
- [ ] 主角的对话和内心独白中，没有提前引用本章尚未出现的信息
- [ ] 其他角色若知道主角有玉佩/铜钱，有明确信息来源

---

## 第五步：检查与前后章的衔接

- [ ] 时间线是否连贯？
- [ ] 上一章结尾的状态/位置/信息，本章开头是否自然承接？
- [ ] 本章结尾给下一章留的钩子是否清晰？
- [ ] 与相邻章节的追兵/冲突类型是否重复？

---

## 完成确认

完成以上五步后，在本地做一个标记文件：

```bash
touch .chapter_ready
```

开始写作。写作完成后进入 `scripts/chapter_checklist.md` 进行交付前检查。
