# 《岁蚀》写作规范

## 写作前 · 必读文档与清单

**硬性规定**：在开始写或修改任何一章**之前**，必须先完成 `scripts/pre_writing_checklist.md` 中的五步准备，并重新通读一次 `docs/笔法核心规则.md`。未完成不得动笔。

### 必读设定文档（按优先级）

1. `docs/笔法核心规则.md` — **每次开写前必须重读**
2. `docs/重构设定_岁蚀全本总纲.md` — 世界观、三条主线、七卷结构
3. `docs/人物档案_重构.md` — 角色设定、阵营、关系
4. `docs/势力划分与部门_重构.md` — 七大势力、部门、出场预设
5. `docs/域外设定与过渡设计.md` — 域外规则、过渡层级
6. `docs/功法与战斗体系.md` — 境界、招式、法宝、阵法
7. `docs/伏笔与线索追踪.md` — 已埋伏笔状态、本章该回收的条目

如果是修改已有章节，还要额外读取该章节原文和前后各一章。

### 开写前必须明确

用三句话写在章节开头的临时注释里：

```
<!--
本章推动：__________
本章改变：__________
本章埋下：__________
章末钩子：__________
本章回收伏笔（来自第X章）：__________
-->
```

如果答不上来"推动了什么、改变了什么、埋了什么"，先回去做大纲，不要动笔。

## 写作核心原则（根据第1-19章经验教训）

1. **人物不定性、不定型**：不提前给角色贴标签。角色的立场、阵营、动机通过选择和行动逐步展露，让读者猜不透。

2. **大纲只是初步设定**：不固化。随情节展开可调整、推翻、重构。如果写作中发现更精彩的方向，改大纲而不是硬塞。

3. **悬念驱动，不信息投喂**：不在同一章内倒完一个设定。切碎成3-5块，借不同角色、在不同情境、隔几章释放一次。

4. **多线交织**：适时切到其他角色POV（沈见微、叶知秋、纪无咎、红姑等），让读者知道主角不知道的信息。

5. **章末必须有钩子**：突然出现的人、未说完的话、反常的细节、悬而未决的决定。不允许"睡着了"/"继续走"/"内心总结"式收尾。

6. **冲突前置**：每一章至少有一个角色在冒风险、做不可逆选择。无冲突的纯信息章不写。

7. **高潮密度**：每8-12章一个小高潮（单挑/突围/反转），跨多章一个大战。

## 写作后 · 交付前检查

写完或改完一章后，必须按顺序执行：

### 1. 运行自动检查脚本

```bash
python scripts/lint_chapter.py chapters/新_第XX章_标题.md
```

必须全绿。任何标红项都要修到通过。

### 2. 逐项过人工检查清单

打开 `scripts/chapter_checklist.md`，逐项勾选。通不过的项目要修改或明确标注原因。

### 3. 同步线索文档与数据库

- 更新 `docs/伏笔与线索追踪.md`：新抛出的线索、回收的线索、新人物/地点/设定
- 执行 `cd server && node sync_chapter.js <章节号>`
- 提交时包含 `server/writer.db`
